## Project overview

**Dictee** is a French dictation learning app for iPhone and iPad (iOS 17+).
Pupils photograph a teacher's paper word list, then practise spelling through typed dictation sessions.

Full spec: [`SPEC.md`](SPEC.md) â€” keep it in sync whenever behaviour changes (see rule below).

## Build system

The Xcode project is **generated by xcodegen** from `project.yml`.
Never edit `Dictee.xcodeproj` by hand â€” all structural changes go in `project.yml`.

```bash

  -scheme Dictee \
  -sdk iphonesimulator \
  -destination 'platform=iOS Simulator,name=iPhone 16' \
  -configuration Debug \

  CODE_SIGNING_REQUIRED=NO \
```

## Architecture & framework rules

| Topic | Rule |
|---|---|
| Reactivity | Use `@Observable` (Observation framework, iOS 17). **Never** use `@ObservableObject` / `@Published`. |
| Instantiating observable services | `@State private var speech = SpeechService()` â€” not `@StateObject`. |
| SwiftData queries | `@Query` only. Never call `modelContext.fetch` manually for view data. |
| N+1 queries | Fetch the full collection once in the parent view and compute per-item values with a helper (see `HomeView.lastScore(for:)`). Do **not** issue a separate `@Query` per list card. |
| Concurrency | Service classes are `@MainActor` by default. Delegate callbacks that arrive off-main must dispatch back with `Task { @MainActor in ... }`. |

## String comparison / answer checking

All answer checking **must** go through `String.normalizedForDictation`:

```swift
// Dictee/Extensions/String+Normalization.swift
var normalizedForDictation: String {
    let apostropheVariants: [Character] = ["\u{2019}", "\u{2018}", "\u{02BC}"]
    let folded = apostropheVariants.reduce(self) {
        $0.replacingOccurrences(of: String($1), with: "'")
    }
    return folded.trimmingCharacters(in: .whitespaces).lowercased()
}
```

Rules enforced by this normalizer:
- **Case-insensitive** â€” "Ă‰cole" matches "Ă©cole".
- **Accent-sensitive** â€” "ecole" does NOT match "Ă©cole".
- **Apostrophe variants folded** â€” iOS Smart Punctuation types U+2019 (curly); stored words use U+0027 (straight). Both sides are normalised before comparison.
- **Whitespace trimmed** â€” leading/trailing spaces are stripped so accidental spaces never fail a correct answer.

Never compare answer strings with `==` or `.lowercased()` alone.

## Keyboard assistance

During sessions the keyboard **must** have all assistance disabled. Use `DictationTextField` (a `UIViewRepresentable` wrapper) instead of the standard SwiftUI `TextField`:

```
Dictee/Views/Session/DictationTextField.swift
```

It disables: `autocorrectionType`, `spellCheckingType`, `smartQuotesType`,
`smartDashesType`, `smartInsertDeleteType`, and clears the `inputAssistantItem`
bar button groups (removes the QuickType bar).

## Data model

| Model | Key fields |
|---|---|
| `WordList` | id, name, createdAt, photoData (JPEG), lastPracticedAt, handwritingNeatness (Double?), words[] (cascade delete) |
| `Word` | id, text, list (WordList?) |
| `ReviewBankEntry` | id, wordId, wordText *(denormalized â€” survives list deletion)*, addedAt, missCount |
| `SessionResult` | id, listId (UUID?), listName, date, isRevisit, isPaperSession, answers[] |
| `Answer` | id, wordId, wordText, typed, correct, session |

`ReviewBankEntry.wordText` is denormalized intentionally â€” entries must survive
if the parent `WordList` is deleted. Do not replace it with a foreign key.
